## 关键组件设计

### 1. 任务确认卡片 (Task Confirmation Card)

```typescript
interface TaskConfirmationCard {
  taskId: string;
  taskName: string;
  
  // 基本信息
  operation: {
    type: string;                    // 'rewrite' | 'generate' | ...
    skillUsed: string[];
    toolsUsed: string[];
  };
  
  // 对象信息
  targetObjects: {
    type: string;                    // 'articles' | 'products' | ...
    count: number;
    ids?: string[];
  };
  
  // 参数配置（可编辑）
  parameters: Record<string, unknown>;
  
  // 执行估计
  estimation: {
    estimatedDurationMs: number;
    estimatedTokens: number;
    estimatedSuccessRate: number;
  };
  
  // 效果预览
  preview?: {
    sampleIndex: number;              // 第几个样本
    before: string;
    after: string;
  };
  
  // 用户操作
  actions: {
    execute: () => void;
    editParameters: () => void;
    cancel: () => void;
  };
}
```

### 2. 任务列表 (Task List Panel - 右侧)

```typescript
interface TaskListPanel {
  tasks: {
    executing: Task[];                // 执行中的任务

  // 每个任务显示的信息
  taskCard: {
    taskId: string;
    taskName: string;
    status: 'executing' | 'completed' | 'failed';
    
    // 执行中
    progress?: number;                // 0-100%
    currentItem?: number;
    totalItems?: number;
    elapsedTime?: number;
    
    // 已完成
    completedTime?: string;
    successCount?: number;
    failureCount?: number;
    
    // 失败
    errorMessage?: string;
    failedAt?: string;
    
    // 操作
    actions: string[];               // ['查看详情', '下载', '重试', ...]
  };
}
```


```
         ↓

         ↓
生成任务确认卡片 (TaskConfirmationCard)
   ├─ 任务类型识别
```

## AI 补充的智能化

### AI 在确认卡片中的职责

```
1. 意图识别
   • 用户说"改写文章" → 识别为 article_rewrite
   • 用户说"帮我翻译" → 识别为 translation
   • 用户说"优化描述" → 识别为 description_optimization

2. 参数提取
   • 从对象选择 → 确定操作范围
   • 从风格参考 → 提取目标风格特征
   • 从约束条件 → 获取限制条件
   • 从自然语言 → 补充或覆盖参数
   
   例: 用户说"简洁一些" + 约束"500字以内"
       → 系统自动设置 maxLength: 500, style: 'concise'

3. Skill/Tool 推荐
   • 识别任务所需的 Skill
   • 推荐相关的 Tool（如质检工具、优化工具等）
   • 考虑执行顺序

4. 执行估计
   • 根据对象数量估计耗时
   • 根据操作复杂度估计 token 消耗
   • 基于历史数据估计成功率

5. 效果预览
   • 抽取 1-3 个样本对象
   • 进行小规模预执行
   • 展示处理前后的对比
   • 让用户直观看到效果
```

## 关键交互优化

### 参数编辑流程

```
用户点 [编辑参数]
         ↓
参数面板展开 (可折叠分组)
   ├─ 基础参数 (自动展开)
   │  ├─ 目标长度
   │  ├─ 风格/语言
   │  └─ 保留内容
   ├─ 高级参数 (可折叠)
   │  ├─ 质检规则
   │  ├─ 优化选项
   │  └─ 自定义规则
   └─ 预设模板
      ├─ 快速预设按钮
      └─ 保存当前配置

用户修改后:
   [保存] [取消] [恢复默认]
   ↓
点 [保存] → 参数生效，显示新的预览
点 [取消] → 回到之前的参数
```

### 预览展示

```
预览分为两部分:

1. 快速预览 (卡片内)
   • 1-3 个样本对比
   • 用户快速评估效果

2. 详细预览 (弹窗)
   用户点 [查看全部预览] 后:
   • 显示全部对象的预处理结果
   • 支持筛选/搜索
   • 支持逐个审核
   • 可以调整个别对象的参数
   • 最后确认执行
```
