# Spark 项目说明文档

## 1. 项目目标
- 本项目是 Shopify 应用，核心是提供 AI Assistant 能力。
- 主要场景包括：店铺经营分析、广告与物流授权引导、运营文案与活动建议、常见业务问答。

## 2. 技术栈与运行形态
- 前端框架：React + TypeScript
- 路由体系：React Router
- UI 组件：Shopify 组件（`s-*` 组件）
- 应用形态：嵌入式 Shopify App（Admin 内使用）

## 3. 关键目录结构
- `app/routes/page/`：页面级路由与页面逻辑
- `app/routes/component/`：聊天、输入、消息等可复用组件
- `app/routes/app.tsx`：应用导航与外层路由壳
- `.github/workflows/`：CI/CD 与部署流程
- `shopify.app.*.toml`：Shopify 环境配置文件

## 4. 当前核心页面与职责
- `app/routes/page/ChatPage.tsx`
  - 聊天页主逻辑
  - 首条欢迎语、快捷问题、清空会话
  - 广告/物流授权配置入口与弹窗
- `app/routes/component/ChatMessages.tsx`
  - 聊天气泡渲染
  - 用户/助手角色标签显示

## 5. 文案与交互约定
- 角色命名统一使用：`AI Assistant`
- 文案默认中文，表达简洁、可执行、对商家友好
- 欢迎语优先使用分段或列表，避免单段过长
- 常用动作（如授权、提交、清空）按钮文案保持清晰一致

## 6. 编码与改动约束
- 优先沿用现有代码风格与命名方式
- 未明确要求时，不修改以下高风险区域：
  - 鉴权与安全相关逻辑
  - 部署流水线与发布流程
  - 环境配置与密钥处理
- 若改动涉及用户可见文案，注意全局一致性（页面标题、导航、气泡标签、提示语）

## 7. 修改前检查清单
- 明确需求是否影响聊天页、消息组件、授权流程
- 先阅读将要修改的文件，再进行最小范围改动
- 避免引入无关重构

## 8. 修改后检查清单
- 运行并检查相关页面交互是否正常
- 不引入新的 lint 错误
- 确认文案一致性与可读性
