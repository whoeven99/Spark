# AI 工作台设计修正 - 正确理解

## 🎯 核心定位修正

### 之前的错误理解
- ❌ ToolBar 是左侧导航栏式面板
- ❌ ToolBar 是顶部水平工具栏

### 正确的理解
✅ **AI 工作台 = 工具集合 + 对话流程**
- 帮助商家实现不同维度的商店运营操作
- 通过对话驱动生成任务
- 支持人工审核和应用

✅ **ToolBar = 对话框下方的工具栏**
- 类似 Claude 对话框下方的工具入口
- 用户在发送对话前补充上下文信息
- 包括：选对象、参考文档、多媒体、约束条件

✅ **任务确认卡片 = 任务创建前的表单摘要**
- 就是任务详情页的一个特殊形态
- 用户可以查看和编辑参数
- 点击启动任务
- 任务状态会与任务列表同步

## 🔄 完整的工作流程

```
STEP 1: 准备上下文
└─> 用户操作 ToolBar:
    • [选对象] → 选择 42 个商品
    • [参考文档] → 上传品牌指南
    • [多媒体] → 上传 3 张参考图
    • [约束] → 设置最大 500 字
    └─> ToolBar 显示: "已补充 4 项"

STEP 2: 发送对话
└─> 用户在输入框输入:
    "帮我改写这些商品描述，符合品牌声音，简洁一点"
    └─> 点 [发送]

STEP 3: AI 理解和补充
└─> AI 处理完整上下文:
    {
      message: "帮我改写...",
      context: {
        selectedObjects: { count: 42, type: 'products' },
        referenceDocuments: [{ 品牌指南 }],
        mediaFiles: [3 张图],
        constraints: { maxLength: 500 }
      }
    }
    └─> AI 返回建议任务

STEP 4: 任务确认卡片展示
└─> 在对话框中展示 TaskConfirmationCard:
    • 任务名称、描述
    • 参数配置（可编辑）
    • 效果预览（样本对比）
    • [执行任务] 主按钮

STEP 5: 用户执行任务
└─> 用户点 [执行任务]
    ├─> 创建真实任务
    ├─> 加入右侧任务列表
    ├─> 对话卡片状态变为"执行中"
    └─> 开始轮询任务进度

STEP 6: 任务执行和状态同步
└─> 任务列表和对话卡片状态同步:
    • 对话卡片显示: "⏳ 执行中 50%"
    • 任务列表显示: 进度条、百分比
    • 所有状态一致

STEP 7: 任务完成
└─> 任务完成后:
    • 对话卡片显示: "✓ 已完成"
    • 任务列表展示最终结果
    • 用户可点击查看详情或应用结果
```

## 🎨 UI 组件关系

### ChatPanel
- 显示对话历史（用户 + AI 消息）
- 顶部是对话历史
- 底部是输入框 (在 ToolBar 上方)

### ToolBar
- 对话框下方的工具栏
- 四个按钮: [选对象] [参考文档] [多媒体] [约束]
- 显示已补充信息数量
- 包含清空功能

### ChatPanel Input
- 在 ToolBar 下方
- 输入框 + [发送] 按钮

### TaskConfirmationCard
- 在对话历史中展示（不是在 ToolBar 上方）
- 作为一个"消息"展示在对话框中
- 包含任务摘要和 [执行任务] 按钮
- 会随着任务状态更新而变化

### TaskListPanel (右侧)
- 固定宽度 340px
- 显示任务队列
- 支持点击查看任务详情
- 状态与对话卡片同步

## 🎯 关键差异说明

| 方面 | 之前 | 现在 |
|------|------|------|
| **ToolBar 位置** | 左侧面板 或 顶部水平 | 对话框下方 |
| **ToolBar 样式** | 卡片式 或 按钮组 | 紧凑的工具栏 |
| **任务确认卡片** | 模态弹窗 | 对话框中的消息 |
| **卡片到任务列表** | 独立流程 | 状态同步 |
| **用户心智** | "在对话旁边补充信息" | "在对话框中补充信息" |

---

## ✨ 设计优势

1. **信息流清晰** - ToolBar 是对话的一部分，不是并列的面板
2. **交互自然** - 类似 Claude，用户习惯这种模式
3. **上下文完整** - 所有信息都在对话框内完成补充
4. **状态同步** - 任务卡片和任务列表保持一致
5. **空间利用** - 中央对话区充分利用空间，右侧任务列表独立

```
    ↓

    ↓

    ↓
创建 TaskConfirmationCard 显示在对话中
    ↓

    ↓

    ↓

    ↓
任务完成，对话卡片显示最终状态
```
