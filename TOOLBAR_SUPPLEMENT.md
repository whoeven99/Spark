# 顶部工具栏补充分析

## 现有工具（4种选择方式）
- [直接选] - Object Picker
- [标签] - Tag Filter  
- [条件] - Advanced Filter
- [CSV] - CSV Upload

---

## 思考：还需要什么？

### 问题诊断

从电商工作流来看，用户可能遇到的场景：

**场景 1**：「我上次选的那些库存预警商品，这次又要用」
- 需要：**快速复用历史选择** 或 **保存为模板**

**场景 2**：「我想看一下选了哪 168 个商品，也想在列表里再调整一下」
- 需要：**预览已选对象的详细列表**，且可以在列表中编辑选择

**场景 3**：「我需要同时满足两个条件的商品：库存<10 AND 价格>100元」
- 现在的设计：用 Advanced Filter 可以做到
- 但如果用户用了多种方式（标签 + 条件），如何组合？
- 需要：**多条件交集/并集操作**

**场景 4**：「这 168 个商品，我想按销量排序，然后只取前 50 个」
- 需要：**排序和截断**（TOP N）

**场景 5**：「我想知道选中这些商品的平均价格、总库存是多少」
- 需要：**快速统计/摘要信息**

**场景 6**：「我想快速选择'销量最好的 100 个'或'新上架的'」
- 需要：**快速预设**（常见的选择快捷方式）

**场景 7**：「我刚才选错了，想撤销上一步」
- 需要：**撤销/重做**

**场景 8**：「我想从已选的 168 个中，再搜索一下包含'红色'的」
- 需要：**已选对象的二次搜索**

## 必需 vs 可选分析

### 🔴 高价值的补充工具（建议加）

#### 1. **预览已选** (Preview Selected)
```
为什么需要：
- 用户对"选了什么"要有可见性
- 可能需要在已选列表中微调（删除某个、添加某个）
- 看统计信息（总数、总价、库存总额等）

UI 示意：
┌─────────────────────────┐
│ [预览] ← 点击显示弹窗    │
│                         │
│ 已选: 168 个商品        │
│ 📊 统计:                │
│ • 总库存: 2,345 件      │
│ • 平均价: ¥156          │
│ • 价格范: ¥29-¥899      │
│                         │
│ 📋 列表 (可搜索/删除):  │
│ ┌──────────────────────┐
│ │ SKU-001 红色T恤      │ ✕
│ │ SKU-002 蓝色连衣裙  │ ✕
│ │ [搜索: ___] 找不到   │
│ │ ...                  │
│ └──────────────────────┘
└─────────────────────────┘
```

#### 2. **快速预设** (Quick Presets)
```
为什么需要：
- 电商中有常见的选择场景
- 一键快速选择，不用每次都重新筛选

示意：
[新上架] [库存预警] [热销TOP50] [待处理] [...]

或下拉菜单：
[快速选择 ▼]
├─ 最近7天新上架
├─ 库存<5件预警
├─ 销量TOP50
├─ 评分<4.0的
├─ 待补充描述
└─ 自定义...
```

#### 3. **保存为模板** (Save as Template)
```
为什么需要：
- "我经常要选库存<10的商品，每次都要重新设置条件"
- 保存筛选条件，下次一键应用
- 类似 SQL 里的"保存查询"

UI：
┌─────────────────────┐
│ 当前条件:           │
│ 库存<10 AND 价格>50 │
│                     │
│ [💾 保存为模板]     │
│ 输入名称: [库存预警]│
│ [保存]              │
└─────────────────────┘

下次使用：
[保存的模板 ▼]
├─ 库存预警
├─ 热销TOP100
└─ 最近上新
```

#### 4. **排序** (Sort)
```
为什么需要：
- 选完对象后，可能需要排序
- "我要优化销量最好的 50 个"
- "我要检查价格最高的 20 个"

UI：
[排序 ▼]
├─ 销量（从高到低）
├─ 价格（从高到低）
├─ 库存（从低到高）
├─ 评分（从高到低）
├─ 最近更新
└─ 自定义字段
```

#### 5. **数据统计摘要** (Statistics)
```
为什么需要：
- 用户想快速了解已选对象的特征
- 帮助做决策

UI：
[📊 统计]
└─ 弹窗显示:
   • 总数: 168
   • 平均价: ¥156
   • 价格范: ¥29-¥899
   • 总库存: 2,345 件
   • 平均评分: 4.2
   • 分类分布: [分类1: 45件] [分类2: 78件]...
```

---

### 🟡 可选的增强工具（后期）

#### 6. **组合/交集** (Combine)
```
使用场景（较复杂）：
"我要选既在'库存预警'标签里，又满足'价格>100'条件的商品"

现状：用户要用Advanced Filter来做
增强：支持可视化的"添加多个条件"
```

#### 7. **撤销/重做** (Undo/Redo)
```
使用场景：
"我刚才选错了，想回到上一步"

UI：
[↶ 撤销] [↷ 重做]
```

#### 8. **导出/导入** (Export/Import)
```
使用场景：
- 导出已选商品列表（可分享或用其他工具）
- 导入预定义的商品列表

UI：
[⬇️ 导出] [⬆️ 导入]
```

#### 9. **对比** (Compare)
```
使用场景：
"我想对比上次选的商品和这次选的有什么区别"

较少用，可后期加
```

## 推荐的最终工具栏设计

### 方案 A：紧凑式（必需功能）
```
┌──────────────────────────────────────────────────────┐
│ [直接选] [标签] [条件] [CSV] │ [快速预设 ▼] │ [预览] │ [统计]
│ ────────────────────────────────────────────────────
│ 已选: 168 件 | [排序 ▼] [保存模板] [清空]
└──────────────────────────────────────────────────────┘
```

### 方案 B：分层式（紧凑 + 隐藏高级）
```
┌────────────────────────────────────────────────┐
│ 第一行（主选择方式）:
│ [直接选] [标签] [条件] [CSV]
│ 已选: 168 件 | [预览] [统计] [⋯ 更多]
│
│ 第二行（可选显示）:
│ [快速预设 ▼] [排序 ▼] [保存模板] [↶ 撤销]
└────────────────────────────────────────────────┘
```

### 方案 C：组织式（分功能组）
```
┌──────────────────────────────────────────────────────┐
│ 📋 选择方式:
│ [直接选] [标签] [条件] [CSV] [快速预设 ▼]
│
│ 📊 管理已选:
│ 已选: 168 件 | [预览] [统计] | [排序 ▼] [保存模板]
│
│ 🔧 操作:
│ [清空] [↶ 撤销] [⋯ 高级]
└──────────────────────────────────────────────────────┘
```

## 我的建议

### 必需（Phase 1）：
1. ✅ 直接选、标签、条件、CSV（已有）
2. ✅ **预览已选** - 用户必须看得见选了什么
3. ✅ **快速预设** - 提高易用性，减少重复操作
4. ✅ **数据统计** - 让用户快速了解选中对象的特性
5. ✅ **排序** - 常见需求（选销量最好的 N 个）
6. ✅ **保存为模板** - 提高工作效率

### 可选（Phase 2）：
7. 撤销/重做
8. 导出/导入
9. 组合/交集（如果Advanced Filter不够）

## 工具栏最终设计建议

### 布局
```
┌─────────────────────────────────────────────────────────────┐
│ AI 工作台 - 选择对象                                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  选择方式: [直接选] [标签] [条件] [CSV] [快速预设 ▼]       │
│                                                             │
│  已选: 168 个商品                                            │
│  统计: 库存总额 2,345 件 | 平均价 ¥156 | 评分 4.2⭐      │
│                                                             │
│  操作: [排序 ▼] [预览清单] [保存为模板] [清空] [↶撤销]    │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ 💬 现在描述你的任务...                                       │
│ [输入框] [发送]                                              │
└─────────────────────────────────────────────────────────────┘
```

### 关键点
1. **信息分组明确**：选择方式 vs 已选统计 vs 操作
2. **必需信息前置**：马上看到"选了多少"和"基本统计"
3. **操作入口清晰**：排序、预览、保存模板等都在一行
4. **渐进式披露**：高级操作可以折叠

## 数据结构补充

```typescript
// 顶部工具栏需要追踪的状态
type SelectionState = {
  // 已选对象
  selectedObjects: {
    type: 'products' | 'articles' | ...;
    ids: string[];
    count: number;
  };
  
  // 选择历史（支持撤销）
  history: SelectionSnapshot[];
  historyIndex: number;
  
  // 保存的模板
  savedTemplates: Array<{
    name: string;
    criteria: FilterCriteria;
    createdAt: string;
  }>;
  
  // 当前排序
  sortBy?: 'sales' | 'price' | 'stock' | ...;
  sortOrder?: 'asc' | 'desc';
  
  // 统计信息（实时计算或缓存）
  statistics?: {
    totalCount: number;
    totalStock: number;
    avgPrice: number;
    priceRange: [min, max];
    avgRating: number;
    categoryDistribution: Record<string, number>;
  };
};
```
