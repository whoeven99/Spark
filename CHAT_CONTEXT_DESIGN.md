# 对话工作台 - 上下文和工具卡片设计

## 问题分析

**场景 1**（问题）：
```
用户说：「帮我把新上的 10 个商品描述优化一下」
系统：收到。但... 怎么知道是哪 10 个商品？
       - 需要用户再说：「名字是 xxx, xxx, xxx 的」？
       - 还是让用户点选择器选？
```

**场景 2**（问题）：
```
用户说：「把这批库存不足的商品都补上描述」
系统：好，但「这批」指的什么？库存多少算不足？
```

**核心痛点**：自然语言无法精确定义操作对象集合。需要**对话 + 结构化工具**结合。

## 完整交互流程示意

```
1. 用户在对话框输入
   └─> "帮我优化库存不足的商品描述"

2. 系统 LLM 分析意图
   ├─> 识别任务：商品描述优化
   ├─> 识别对象筛选条件：库存不足
   ├─> 识别缺少的信息：具体商品、优化风格、语言等
   └─> 返回"需要确认"的建议

3. 系统在对话下方展示上下文卡片
   ├─ 对象选择工具（标签/筛选/直选/CSV）
   ├─ 参数配置面板
   └─ 当前上下文面板

4. 用户在卡片中操作
   ├─> 点击"按标签选择" → 勾选"库存预警"
   ├─> 配置参数 → 风格改为 SEO、长度改为"中"
   └─> 点预览 → 看样本效果

5. 系统更新对话上下文
   ├─> 对话中显示"已选 168 个商品"
   ├─> 显示参数配置摘要
   └─> 提示"准备好执行了吗？"

6. 用户确认 → 执行任务
   └─> 任务进入执行队列
```

## 关键设计原则

### 1. 明确的上下文切换
```
对话 ←→ 工具操作 ←→ 预览 ←→ 对话

用户不需要记住"我选了什么"，系统始终显示当前上下文
```

### 2. 渐进式披露
```
简单操作 → [点击]标签选择 → [高级]条件筛选
不要一次性展示所有选项，让用户逐步深入
```

### 3. 即时反馈
```
选择完对象 → 立即显示数量和预览
配置参数 → 实时预览效果
```

### 4. 快速路径 vs 完整路径
```
快速路径：对话 → 一键应用 → 执行
完整路径：对话 → 细粒度选择 → 参数调优 → 预览 → 执行
```

## 示例应用场景

### 场景 1：快速优化
```
用户: 「优化销量最高的 10 个商品描述」

系统: 
  1. 自动按销量排序，选中 Top 10
  2. 显示预设"专业风格"参数
  3. 提示"确认执行吗？"
  4. 用户一键执行
  
总耗时: 10 秒
```

### 场景 2：精细控制
```
用户: 「帮我把这个 Excel 表里的 200 个商品都翻译成日文」

系统:
  1. 用户上传 Excel
  2. 系统检测列名，映射 SKU 和商品名
  3. 用户配置"日文翻译"参数
  4. 预览 3 个样本
  5. 确认执行
  
总耗时: 2-3 分钟（但非常精确）
```

### 场景 3：条件自动化
```
用户: 「只要库存低于 10 件的商品，就自动补充描述」

系统:
  1. 用户设置筛选条件：库存 < 10
  2. 配置描述参数
  3. 选择"每天自动执行" → 沉淀为 Playbook
  4. 系统每天自动执行
  
总耗时: 一次 5 分钟设置，之后自动
```
